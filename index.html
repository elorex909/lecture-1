Memory Hierarchy

Storage Performance Pyramid

Computer systems use a hierarchy of storage types to balance speed, cost, and capacity.

// Memory Hierarchy (Fastest to Slowest)
1. CPU Registers (1-2 nanoseconds)
2. Cache Memory (~10 nanoseconds)
3. Main Memory/RAM (~100 nanoseconds)
4. Solid State Drives (~100 microseconds)
5. Hard Disk Drives (~10 milliseconds)
6. Magnetic Tapes (seconds)

Each level trades speed for capacity and cost efficiency, with dramatic performance differences between levels.

Visual Analogy:

If accessing RAM is like checking a book's index (taking 20 seconds), then accessing a magnetic disk is like going to the library to get the physical book (taking 58 days).

// Performance Scale
1 second = 1,000 milliseconds
1 millisecond = 1,000,000 nanoseconds
Disk access is ~250,000x slower than RAM!

Course Objective: Minimize disk accesses through efficient file organization and algorithms.
